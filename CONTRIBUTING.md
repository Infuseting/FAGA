## Code de conduite

En participant à ce projet, vous vous engagez à maintenir un environnement accueillant, respectueux et exempt de harcèlement pour tous.

## Comment contribuer ?

### 1. Signaler un bug

Avant de créer une "Issue", assurez-vous que le bug n'a pas déjà été signalé. Si ce n'est pas le cas, ouvrez une issue en incluant :

* Une description claire du problème.
* Les étapes pour reproduire le bug.
* Le comportement attendu vs le comportement actuel.
* Votre version du logiciel/système d'exploitation.

### 2. Proposer une fonctionnalité

Les suggestions sont encouragées ! Ouvrez une issue pour discuter de l'idée avant de commencer à coder, afin de s'assurer qu'elle s'aligne avec la vision du projet.

### 3. Soumettre des changements (Pull Requests)

1. **Forkez** le dépôt.
2. **Créez une branche** pour votre modification (`git checkout -b feature/nom-genial`).
3. **Commitez** vos changements avec des messages clairs.
4. **Pushez** vers votre branche (`git push origin feature/nom-genial`).
5. Ouvrez une **Pull Request**.

## Normes de codage

Pour maintenir la qualité du code, merci de respecter les règles suivantes :

* Suivez le style d'indentation existant (ex: 4 espaces).
* Nommez vos variables et fonctions de manière explicite (en anglais de préférence).
* Documentez les nouvelles fonctions ou les changements complexes.
* Ajoutez des tests unitaires si possible.

## Processus de Pull Request

* Toute PR doit être liée à une issue existante.
* Le projet doit compiler/s'exécuter sans erreur après vos modifications.
* Un mainteneur examinera votre code. Soyez prêt à effectuer des ajustements si nécessaire.
* Une fois approuvée, votre branche sera mergée dans `main`.
